# Web Feature

## 输入约定

输入应尽量包含：

- 功能目标
- 页面或业务流程
- 目标用户和场景
- 相关接口契约
- 是否涉及登录态、权限态、空态、错误态

## 输出约定

输出必须尽量包含：

- 影响页面 / 区块 / 组件
- 交互状态处理要求
- 契约同步点
- 验证要求或验证结果
- 剩余风险

适用场景：

- 用户端页面
- 用户流程
- 内容展示
- 用户操作入口

默认流程：

1. 确认功能归属 `WebServiceWorkSpace`
2. 阅读任务文档和接口契约
3. 按业务流程拆分页面和区块
4. 接入数据和交互
5. 补齐：
   - loading
   - empty
   - error
   - success
   - 未登录 / 无权限（如适用）
6. 运行前端验证

默认不要：

- 套用后台管理组件语义
- 为复用而牺牲用户流程连贯性
