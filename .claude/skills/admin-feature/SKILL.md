# Admin Feature

## 输入约定

输入应尽量包含：

- 功能目标
- 页面或模块名称
- 相关角色和权限要求
- 是否涉及列表、表单、弹窗、抽屉、审核流
- 相关接口契约和任务文档

## 输出约定

输出必须尽量包含：

- 影响页面 / 区块 / 组件
- 权限影响
- 状态处理要求
- 是否涉及审计日志
- 契约同步点
- 验证要求或验证结果
- 剩余风险

适用场景：

- 用户管理
- 角色权限
- 菜单配置
- 审核流
- 系统配置
- 审计日志

默认流程：

1. 确认功能归属 `webAdminServiceWorkSpace`
2. 确认接口契约和权限要求
3. 页面优先拆分：
   - 筛选区
   - 工具栏
   - 数据表格
   - 分页区
   - 编辑弹窗 / 抽屉
4. 补齐状态：
   - loading
   - empty
   - error
   - success
   - 无权限
5. 评估是否需要审计日志

默认不要：

- 把权限只做成按钮隐藏
- 把所有列表、表单、弹窗逻辑堆在一个页面文件里
