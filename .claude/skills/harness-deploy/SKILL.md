# Harness Deploy

## 输入约定

输入应尽量包含：

- 目标环境
- 变更范围
- 发布目标
- 风险等级
- 回滚方式
- 用户确认状态

## 输出约定

输出必须尽量包含：

- 部署目标
- 影响服务
- 风险说明
- 回滚入口
- 执行前检查项
- 执行结果或未执行原因

触发前必须确认：

- 部署目标环境
- 变更范围
- 风险等级
- 回滚方式
- 用户显式确认

执行时必须说明：

- 将要做什么
- 影响哪些服务
- 是否涉及生产
- 回滚入口是什么

禁止：

- 在未确认环境和风险时直接执行
- 把生产部署当作默认动作
