# Agent Runtime Review

## 输入约定

输入应尽量包含：

- Runtime 方案或实现现状
- 权限模型
- 审计模型
- 工具列表
- 预期运行方式
- 已知风险或问题

## 输出约定

输出必须尽量包含：

- 架构问题
- 安全问题
- 可测试性问题
- 可运维性问题
- 优先级排序
- 修正建议

评审时至少检查：

- 是否有明确的计划确认点
- 是否有权限分级
- 是否有审计记录
- 是否限制高风险工具
- 是否可测试
- 是否可回滚
- 是否与业务后端权限边界解耦
