# Fullstack Slice

## 输入约定

输入应尽量包含：

- 任务目标
- 涉及页面、接口或业务对象
- 当前任务文档路径（如存在，优先 `.claude/tasks/*.md`）
- 当前接口契约状态
- 是否涉及数据库、权限、审计和联调

## 输出约定

输出必须尽量包含：

- 影响范围（前端 / 后端 / 契约 / 数据）
- 后端改动点
- 前端改动点
- 契约同步点
- 验证要求或验证结果
- 剩余风险

使用本 Skill 时，默认按以下顺序执行：

1. 阅读任务文档和接口契约
2. 判断影响范围：
   - `WebServiceWorkSpace`
   - `webAdminServiceWorkSpace`
   - `bootServiceWorkSpace`
3. 先补或确认接口契约
4. 先实现后端边界：
   - 参数校验
   - 权限校验
   - 事务 / 幂等
   - 结构化错误
5. 再实现前端：
   - 页面结构
   - 数据接入
   - loading / empty / error / success
6. 运行匹配的验证
7. 输出改动摘要、验证结果、剩余风险

默认禁止：

- 只改前端不确认后端契约
- 只改后端不说明前端影响
- 跳过联调影响分析
