# Harness Diagnose

## 输入约定

输入应尽量包含：

- 失败现象
- 错误日志或报错信息
- 触发环境
- 最近变更范围
- 是否为生产问题

## 输出约定

输出必须尽量包含：

- 现象描述
- 根因判断
- 影响范围
- 修复建议
- 是否需要进一步数据
- 风险等级

默认流程：

1. 先收集日志、构建结果、错误信息
2. 区分：
   - 配置问题
   - 环境问题
   - 权限问题
   - 构建问题
   - 部署问题
3. 输出：
   - 现象
   - 根因判断
   - 影响范围
   - 修复建议

默认不要：

- 在没有明确授权时触发生产操作
