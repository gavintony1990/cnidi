# 0001-项目初始化

## 背景

天工联项目以 `claude-vibe-framework` 为基础启动，需要将脚手架收口成真实业务项目代码库。

## 目标

- 项目英文名收口为 `cnidi`
- 项目中文名收口为“天工联项目”
- 保留三端最小骨架作为后续开发起点
- 保留治理层、规则、技能和工作流，作为真实业务项目约束

## 影响范围

- 根目录文档
- `.claude` 治理层
- `WebServiceWorkSpace`
- `webAdminServiceWorkSpace`
- `bootServiceWorkSpace`

## 当前范围

- 不实现具体业务模块
- 不补数据库表和业务接口
- 不接入真实部署环境

## 技术栈确认

- 用户端：React + Vite
- 管理后台：React + Vite + Ant Design
- 后端：Spring Boot + Maven 多模块

## 第一里程碑

- 完成项目命名收口
- 完成三端最小骨架命名同步
- 完成后端坐标、包名和应用名同步
- 进入首个业务闭环设计阶段
