# 用户登录

管理后台实现账号密码登录，后端提供登录/登出/当前用户接口，前端完成登录页和登录态管理。

## 背景

- 当前三端骨架已就绪，但没有任何认证机制
- 管理后台所有页面均无鉴权保护
- 认证是后续所有业务模块的前置依赖，必须首先打通

## 目标

- 管理后台可以通过账号密码登录，登录后进入系统首页
- 未登录访问任何页面自动跳转到登录页
- 后端 JWT 签发和校验机制就绪，后续接口可直接接入
- 登出后清除 token，跳回登录页

## 影响范围

- `webAdminServiceWorkSpace`：新增登录页、路由守卫、token 存储、请求拦截器
- `bootServiceWorkSpace`：新增登录/登出/当前用户接口、JWT 工具、用户表、密码加密
- `WebServiceWorkSpace`：**不受影响**（用户端认证另行规划）
- `.claude`：同步 api-contract.md

## 需求说明

**用户角色**：管理员

**核心场景**：
1. 管理员访问管理后台任意路由 → 未登录则跳转 `/login`
2. 输入账号（用户名或邮箱）+ 密码 → 点击登录
3. 登录成功 → 获取 JWT token，存入 localStorage → 跳转 `/dashboard`（或原目标页）
4. 登录失败 → 展示错误提示（账号或密码错误）
5. 页面内任意请求返回 401 → 清除 token，跳转 `/login`
6. 点击登出 → 调用登出接口 → 清除 token → 跳转 `/login`

**异常场景**：
- 账号不存在：返回统一错误，不区分账号/密码（防枚举）
- 密码错误：同上
- token 过期：401 拦截，前端跳登录页
- 网络异常：前端展示通用错误提示

**权限要求**：登录接口无需鉴权，登出和当前用户接口需携带有效 token

## 接口契约变更

### POST /api/auth/login

- **Auth**：无
- **Request Body**：
  ```json
  { "username": "admin", "password": "123456" }
  ```
- **Response**：
  ```json
  { "data": { "token": "eyJ...", "expiresIn": 86400 }, "requestId": "req_123" }
  ```
- **Error**：
  - `401 UNAUTHORIZED` — 账号或密码错误
  - `400 VALIDATION_ERROR` — 参数缺失或格式错误
- **使用方**：`webAdminServiceWorkSpace`

### POST /api/auth/logout

- **Auth**：Bearer token
- **Request Body**：无
- **Response**：`{ "data": null, "requestId": "req_123" }`
- **使用方**：`webAdminServiceWorkSpace`

### GET /api/auth/me

- **Auth**：Bearer token
- **Response**：
  ```json
  { "data": { "id": 1, "username": "admin", "roles": ["ADMIN"] }, "requestId": "req_123" }
  ```
- **Error**：`401 UNAUTHORIZED` — token 无效或过期
- **使用方**：`webAdminServiceWorkSpace`

## 数据变更

**新增表 `sys_user`**：

```sql
CREATE TABLE sys_user (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT,
  username    VARCHAR(64) NOT NULL UNIQUE,
  email       VARCHAR(128),
  password    VARCHAR(255) NOT NULL,  -- BCrypt hash
  status      TINYINT(1) NOT NULL DEFAULT 1,  -- 1启用 0禁用
  created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  created_by  BIGINT,
  updated_by  BIGINT
);
```

初始数据：插入一条默认管理员账号（username: admin，密码明文在 SQL 注释中说明，存储为 BCrypt hash）。

**迁移方式**：Flyway 脚本 `V001__create_sys_user.sql`

**回滚方式**：`DROP TABLE sys_user`

## 前端任务

### 用户端

不受影响。

### 管理后台（webAdminServiceWorkSpace）

- **路由**：引入 React Router，配置 `/login` 和 `/dashboard`（当前首页）
- **路由守卫**：`PrivateRoute` 组件，无 token 跳 `/login`
- **登录页** `LoginPage`：账号 + 密码输入框、登录按钮、loading 态、错误提示
- **token 管理**：封装 `tokenStorage`（localStorage），存/取/清
- **请求封装**：axios 实例，统一设置 `Authorization: Bearer <token>`，401 拦截跳登录页
- **登出**：顶部导航栏添加登出入口，调用接口后清除 token

## 后端任务

- **模块**：在 `system-biz` 中实现，`system-api` 中定义 DTO
- **Controller**：`AuthController`，三个接口，无鉴权过滤登录接口
- **Service**：`AuthService`，用户查询 + BCrypt 密码校验 + JWT 签发
- **Repository**：`SysUserMapper`，按 username 查用户
- **JWT 工具**：`JwtUtils`，签发/解析/校验，放 `common-core`
- **Security 配置**：Spring Security 基础配置，放行 `/api/auth/login`，其余接口需 token
- **统一异常**：`UNAUTHORIZED` 错误码对应 401
- **权限校验**：`/api/auth/me` 从 token 中解析用户信息返回

## 测试点

- 正确账号密码 → 登录成功，返回 token
- 错误密码 → 401，前端显示提示
- 不存在账号 → 401，同上（不泄露账号是否存在）
- 禁用账号 → 401，提示账号已禁用
- 携带有效 token 访问 `/api/auth/me` → 返回用户信息
- token 过期或伪造 → 401
- 前端未登录直接访问 `/dashboard` → 跳转 `/login`
- 登录后访问 `/login` → 跳转 `/dashboard`
- 登出后 token 清除，再访问需鉴权页面跳登录页

## 验收标准

- 管理后台登录流程完整可运行（登录 → 首页 → 登出）
- 路由守卫生效，未登录无法访问受保护页面
- 接口契约已同步到 `api-contract.md`
- 后端接口已通过基础测试（至少 mvn test + 手动验证）
- 前端已通过 lint + typecheck + build
- 登录页处理 loading / error / success 三态

## 风险与待确认项

- **数据库**：当前骨架未配置数据库连接，需在实现前确认 MySQL/PostgreSQL 选型和连接配置
- **JWT secret**：开发阶段写死配置，生产需通过环境变量注入，不提交到代码
- **token 刷新**：本期不做 refresh token，token 过期直接重新登录
- **用户端登录**：本期只做管理后台，用户端认证另行规划
