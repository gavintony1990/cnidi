# Tasks Guide

本目录是项目需求定义目录，所有需求的入口层统一放在这里。

## 作用

- 作为单次需求的唯一入口
- 给 rules、skills、agents 提供统一上下文
- 作为从需求拆解到开发、测试、入库的起点

## 使用规则

- 一个需求对应一个任务文档
- 文件名建议使用：

```txt
0001-需求名称.md
```

- 新任务优先复制 `0000-task-template.md`
- 任务文档中必须明确：
  - 背景
  - 目标
  - 影响范围
  - 接口契约变更
  - 数据变更
  - 前后端任务
  - 测试点
  - 验收标准
  - 风险
