# Java Spring Boot Backend Rules

本文件定义 Spring Boot 后端实现规则，适用于当前项目的 `bootServiceWorkSpace`。

## 工作前检查

开始实现前必须确认：

- 已明确当前需求涉及哪些模块。
- 已确认接口契约是否存在或需要补充。
- 已确认权限、事务、审计、数据库影响和测试影响。

## 分层职责

推荐分层：

```txt
controller/
application/ 或 service/
domain/（如项目采用）
repository/ 或 mapper/
config/
security/
common/
```

职责要求：

- `Controller`：参数接收、校验触发、调用应用层、返回统一响应。
- `Application/Service`：业务编排、权限决策、事务边界、状态流转。
- `Repository/Mapper`：数据访问，不写业务规则。
- `Config`：Bean、配置、外部资源初始化。
- `Security`：认证、授权、用户上下文、权限表达式。

## Controller 规则

- Controller 不直接访问 Mapper/Repository。
- Controller 不处理复杂业务判断。
- 输入对象使用 DTO / Command，必须有参数校验。
- 返回对象使用统一响应结构，不直接暴露内部异常。
- 查询、创建、修改、删除接口语义要稳定，不混用。

## Service 规则

- Service 是事务边界的主要承载层。
- Service 负责核心状态变更、幂等、权限决策和审计触发。
- 涉及多个 repository 的写操作，要明确事务边界。
- 跨模块调用优先依赖稳定接口，不直接穿透到底层实现。

## Repository / Mapper 规则

- 只处理数据访问，不做权限判断和业务编排。
- SQL 或条件构造必须明确分页、排序、筛选边界。
- 查询接口避免无界扫描和 N+1。
- 数据访问层不要返回前端展示对象。

## 校验与统一响应

- 参数校验使用 `jakarta.validation` / `javax.validation` 注解或等价机制。
- 校验失败、业务失败、权限失败都必须转换为统一结构化错误。
- 统一成功响应和错误响应应与 `.claude/docs/api-contract.md` 对齐。

## 权限与安全

- 后端是最终权限边界。
- 管理后台敏感操作必须进行权限校验和必要的审计记录。
- 不信任前端传入的角色、权限、状态和价格等关键字段。
- 文件上传、导出、批量操作必须考虑权限、大小限制和审计需求。

## 事务与幂等

- 写操作默认评估事务边界。
- 重试、补偿、重复提交要考虑幂等。
- 状态流转型接口必须防止非法状态跳转。
- 批量操作要明确部分成功、部分失败和回滚策略。

## 异常与日志

- 业务异常使用统一异常类型和错误码。
- 全局异常处理统一转换响应。
- 错误日志要包含请求标识、用户标识、对象标识和失败原因。
- 不在 Controller 层到处 `try/catch` 吞异常。

## 测试与验证

至少按改动范围执行：

```bash
mvn test
mvn package
```

如涉及：

- 接口行为变化：补接口测试或集成测试
- 权限变化：补权限场景测试
- 状态流转：补关键路径测试
- 数据库变化：补迁移和回归说明

如无法运行，必须说明未验证部分和剩余风险。
