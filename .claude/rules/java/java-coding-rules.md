# Java Coding Rules

本文件定义 Java 编码规则，适用于 `bootServiceWorkSpace`。规则参考阿里巴巴 Java 开发手册的常用约束，并结合当前项目的 Spring Boot + Maven 多模块架构做裁剪。

## 工作前检查

开始实现前必须确认：

- 已阅读 `CLAUDE.md`、`.claude/claude.md`、当前任务文档。
- 如涉及接口，已阅读 `.claude/docs/api-contract.md`。
- 已明确当前需求影响的是 controller、service、repository、数据库还是接口契约。

## 命名规则

- 包名全部小写，按职责分层，不使用拼音缩写和无意义缩写。
- 类名使用 UpperCamelCase。
- 方法、变量使用 lowerCamelCase。
- 常量使用 `UPPER_SNAKE_CASE`。
- 抽象类优先使用 `AbstractXxx`，工具类使用 `XxxUtils` 时必须确实无状态。
- 接口命名不强制 `I` 前缀，实现类使用业务语义命名，不使用 `Impl2`、`NewImpl` 这类临时命名。
- DTO、VO、Query、Command、Event 等对象后缀要明确语义，不混用。

## 基础编码规则

- 一个类只承担一类核心职责，避免“大而全”类。
- 方法应保持单一职责；过长方法要拆分，但不要为拆分而拆分。
- 禁止使用魔法值；有业务语义的数字和字符串必须提取为常量或枚举。
- 禁止直接使用 `HashMap<String, Object>` 代替明确的 DTO，除非确实处理动态字段。
- `equals` 比较优先使用常量或确定非空对象在前，避免空指针。
- `BigDecimal` 比较使用 `compareTo`，金额和精度计算禁止用 `double`。
- 集合判空使用清晰方式，避免空指针链式调用。
- 避免过度使用继承，优先组合。

## 对象分层规则

- `Entity`：数据库持久化对象，只承载持久化字段和少量领域内聚行为。
- `DTO` / `Command` / `Query`：接口输入输出和应用层传输对象。
- `VO`：前端展示对象。
- 禁止直接把 `Entity` 返回给前端。
- 禁止 Controller 直接操作 Mapper/Repository。
- 禁止把复杂业务逻辑写进 DTO / VO / Entity 的 getter/setter。

## Optional、枚举与异常

- `Optional` 用于表达返回值可能为空的场景，不用于字段、参数和集合元素类型。
- 枚举必须承载明确业务语义；不要把数据库 code 和展示文案散落在多个地方。
- 禁止直接抛出 `RuntimeException("xxx")` 作为业务错误。
- 业务异常应使用统一异常体系和稳定错误码。

## 日志规则

- 使用统一日志框架，不使用 `System.out.println`。
- 日志必须有业务上下文：请求标识、用户标识、关键对象标识。
- 错误日志记录异常时不要重复打印同一异常多次。
- 不打印密码、token、身份证号、银行卡号等敏感信息。
- 高频接口避免无意义 info 日志刷屏。

## 集合、并发与性能

- 集合初始化容量在已知规模时应显式指定。
- 使用流式 API 时优先保证可读性，避免过度链式处理。
- 并发场景明确线程安全边界，不在共享可变对象上做隐式并发访问。
- 缓存、批量处理和异步逻辑必须说明一致性和失败处理策略。

## 注释与文档

- 注释写“为什么”，不是重复代码字面意思。
- 对公共接口、复杂规则、边界条件和临时兼容逻辑必须有说明。
- 临时兼容代码要标注移除条件，不允许长期无来源保留。

## 禁止行为

- 不允许为了省事关闭校验、权限或事务。
- 不允许直接返回数据库或中间件原始异常给前端。
- 不允许在一个提交里顺手大规模格式化无关文件。
- 不允许引入无必要的重型框架或自定义基础设施。
