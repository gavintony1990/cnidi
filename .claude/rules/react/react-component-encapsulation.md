# React Component Encapsulation Rules

本文件补充 React 组件封装的具体约束。

## 组合优先

- 优先通过组合组件解决复用，而不是通过继承或过度高阶封装
- 当多个页面共享相同区块结构时，优先抽页面区块组件
- 只有在多个页面都需要同一业务语义时，才抽业务组件

## hooks 使用

- 自定义 hook 用于封装可复用状态和副作用
- hook 不承载页面专属的大量 UI 分支逻辑
- hook 返回值保持稳定、清晰，不返回过于庞杂的对象

## 表格与表单

- 表格列定义、筛选条件、批量操作逻辑要集中，不分散在多个子组件
- 表单校验规则、默认值和提交流程要收敛在页面层或表单 hook
- Modal / Drawer 中的编辑表单优先拆成独立业务组件

## 错误封装

- 组件只处理自身能消化的错误状态
- 请求错误、权限错误、全局错误优先在页面层或统一请求层收敛

## 反模式

- “万能业务组件”：一个组件既负责列表、表单、权限、请求和弹窗
- “万能 hook”：一个 hook 返回十几个状态和回调，难以维护
- “万能 props”：通过单个对象传入大量隐式配置
