# Frontend Component Shared Rules

本文件只定义前端组件封装的跨框架共享原则。具体组件组织方式、组合模式和状态封装细节必须继续读取对应框架规则：

- React：`.claude/rules/react/*`
- Vue：`.claude/rules/vue/*`

## 组件封装目标

- 复用真实存在的交互结构
- 保持页面职责清晰
- 降低重复代码
- 避免样式、状态和接口逻辑散落
- 让维护者能快速定位职责

## 跨框架共享边界

组件应该封装：

- 稳定重复的结构
- 可复用交互模式
- 局部 UI 状态
- 通用样式和 design token 使用方式

组件不应该封装：

- 一次性页面结构
- 与单页强耦合的临时逻辑
- 同时处理展示、权限、请求和复杂业务编排的大组件
- 仅为减少少量模板代码而制造的抽象

## 页面与子组件职责

页面负责：

- 路由入口
- 页面级布局
- 数据获取与刷新
- 页面级 loading / empty / error
- 权限入口控制
- 页面级筛选、分页、弹窗、抽屉状态

子组件负责：

- 局部 UI 结构
- 稳定可复用交互
- 明确输入输出的展示行为

## 通用验收标准

- 页面职责和子组件职责清晰
- API 调用没有散落到纯展示组件
- 组件可复用点明确，不是为抽象而抽象
- 页面存在完整状态处理
- 权限控制没有只依赖前端隐藏
