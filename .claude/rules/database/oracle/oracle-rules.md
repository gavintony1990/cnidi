# Oracle Rules

本文件定义 Oracle 场景下的表结构和 SQL 规则。使用 Oracle 时，必须先遵守 `../sql-general-rules.md`，再遵守本文件。

## 建表规则

- 表名、字段名、序列、索引命名必须统一规范。
- 金额和精度字段使用 `NUMBER(p, s)` 明确精度。
- 时间字段根据项目统一约定选择 `DATE` 或 `TIMESTAMP`。
- 需要主键生成策略时，明确序列、自增兼容方案或应用层 ID 策略。

## 索引规则

- 索引必须由查询路径驱动设计。
- 高频筛选、排序、关联场景要明确联合索引和执行计划。
- 不为所有字段机械建索引。
- 大表索引变更要评估重建成本和上线窗口。

## SQL 规则

- 分页查询必须明确排序和分页方式。
- 复杂多表查询要优先保证可读性和执行计划稳定。
- 批量写操作必须明确事务边界和失败处理。
- 避免把大量业务规则堆进难以维护的 SQL 片段。

## 事务与并发

- 明确事务边界、锁冲突风险和热点更新风险。
- 对重要状态流转场景，必须说明幂等、重试和补偿策略。
- 高并发写场景要评估唯一约束、版本控制和应用层限流。

## 验证建议

涉及重要 SQL 时建议补充：

- 执行计划检查
- 索引命中验证
- 并发更新验证
- 大数据量查询验证
