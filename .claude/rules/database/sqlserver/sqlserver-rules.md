# SQL Server Rules

本文件定义 SQL Server 场景下的表结构和 SQL 规则。使用 SQL Server 时，必须先遵守 `../sql-general-rules.md`，再遵守本文件。

## 建表规则

- 表名、字段名使用统一命名规范，不混用多套大小写风格。
- 精确金额和比例字段使用 `DECIMAL/NUMERIC`。
- 时间字段统一选择 `datetime2` 或项目既定方案。
- `bit` 字段只承载明确布尔语义。

## 索引规则

- 所有高频查询都要基于查询路径设计索引。
- 聚集索引和非聚集索引要根据主查询路径明确选择。
- 大表新增索引要评估维护成本和发布时间窗口。
- 查询覆盖需求明确时，可考虑包含列，但必须说明服务场景。

## SQL 规则

- 分页必须使用稳定排序和明确分页语义。
- 批量更新、删除必须明确 where 条件。
- 复杂报表和统计查询要评估执行计划、排序、哈希和临时对象开销。
- 避免写强依赖数据库方言技巧但可读性极差的 SQL。

## 事务与并发

- 重要写操作明确锁和并发策略。
- 不做无必要的大事务和长事务。
- 状态变更、批处理、导入导出场景必须说明失败与补偿策略。

## 验证建议

涉及重要 SQL 时建议补充：

- 执行计划检查
- 索引命中验证
- 并发更新验证
- 大数据量分页验证
