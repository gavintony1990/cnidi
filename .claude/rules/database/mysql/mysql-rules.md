# MySQL Rules

本文件定义 MySQL 场景下的表结构和 SQL 规则。使用 MySQL 时，必须先遵守 `../sql-general-rules.md`，再遵守本文件。

## 建表规则

- 优先使用 `InnoDB`。
- 字符集和排序规则必须统一，避免同库同表混乱。
- 表名、字段名使用小写下划线风格。
- 金额、比例、精度字段使用 `DECIMAL`，禁止使用 `FLOAT` / `DOUBLE` 表达精确业务值。
- 时间字段根据业务选择 `DATETIME` 或 `TIMESTAMP`，并统一项目约定。
- 布尔语义字段建议使用 `TINYINT(1)` 或项目统一布尔类型方案。

## 索引规则

- 联合索引顺序必须根据查询 where、order by、limit 的组合设计。
- 高频唯一查询应使用唯一索引，不依赖普通索引 + 代码判断。
- 不在超长 `VARCHAR` 上盲目建索引；必要时评估前缀索引或冗余字段方案。
- 大表加索引要评估锁表、耗时和发布窗口。

## SQL 规则

- 分页查询必须带稳定排序。
- 模糊查询优先评估是否会导致索引失效。
- 批量更新和删除必须显式带条件。
- 复杂统计 SQL 要评估执行计划和临时表/文件排序风险。
- 需要时用 `EXPLAIN` 检查关键 SQL。

## 逻辑删除规则

- 逻辑删除字段建议统一命名，如 `is_deleted`。
- 所有查询必须统一处理逻辑删除条件。
- 唯一索引和逻辑删除组合场景必须提前设计，避免“已删除数据阻止新建”。

## 事务与锁

- 高并发更新场景要明确使用悲观锁、乐观锁或唯一约束兜底。
- 不做无必要的大事务。
- 跨多表状态变更必须明确事务边界和失败回滚策略。

## 验证建议

涉及重要 SQL 时建议补充：

- `EXPLAIN`
- 索引命中验证
- 边界数据量验证
- 事务和并发场景验证
