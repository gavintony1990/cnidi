# PostgreSQL Rules

本文件定义 PostgreSQL 场景下的表结构和 SQL 规则。使用 PostgreSQL 时，必须先遵守 `../sql-general-rules.md`，再遵守本文件。

## 建表规则

- 表名、字段名使用统一的小写下划线命名。
- 精确金额和比例字段使用 `numeric` / `decimal`。
- 时间字段明确是否带时区，整个项目保持一致。
- JSON/JSONB 仅用于确实动态且难以结构化的字段，不替代清晰表结构。
- 枚举实现要在数据库枚举、检查约束、普通 code 字段之间做明确选择，不混乱使用。

## 索引规则

- 高频查询根据 where、order by、join 设计索引。
- 需要模糊检索、数组检索、JSONB 检索时，必须明确索引策略。
- 复杂条件场景可使用部分索引或表达式索引，但必须有明确业务依据。
- 索引设计要同时关注读性能和写入放大。

## SQL 规则

- 分页查询必须有稳定排序。
- 使用 CTE、窗口函数、JSONB 操作时优先保证可读性和执行计划可控。
- 不写无界全表更新、删除。
- 复杂分析 SQL 必须评估成本和数据量。

## 事务与并发

- 明确事务隔离级别需求，不默认依赖数据库“会处理好一切”。
- 高并发状态变更必须评估行锁、唯一约束、乐观并发或应用层幂等。
- 使用 `upsert` 时要说明冲突键和最终语义。

## 验证建议

涉及重要 SQL 时建议补充：

- `EXPLAIN ANALYZE`
- 索引命中验证
- 写放大与热点更新评估
- 并发事务场景验证
