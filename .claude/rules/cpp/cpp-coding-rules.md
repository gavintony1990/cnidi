# C++ Backend Rules

本文件定义 C++ 后端实现规则，适用于项目中存在 C++ 服务或用户明确要求 C++ 方案的场景。

## 工作前检查

开始实现前必须确认：

- 已阅读 `CLAUDE.md`、`.claude/claude.md`、当前任务文档。
- 如涉及接口，已阅读 `.claude/docs/api-contract.md`。
- 已明确当前需求是否真实落在 C++ 工程中，而不是语言迁移讨论。

## 工程与分层

推荐结构：

```txt
include/
src/
  api/
  service/
  repository/
  protocol/
  common/
tests/
bench/
```

职责要求：

- `api`：协议适配、请求解析、响应返回。
- `service`：业务编排、状态流转、权限决策。
- `repository`：数据访问，不写业务流程。
- `protocol`：Protobuf / 自定义协议定义与序列化边界。
- `common`：日志、配置、错误码、基础工具。

## 编码规则

- 优先 RAII、清晰所有权和显式生命周期。
- 明确 `unique_ptr`、`shared_ptr`、`weak_ptr` 使用边界。
- 不用裸指针表达长期所有权。
- 不吞错误，不隐藏失败路径。
- 并发逻辑必须明确锁边界、线程退出和资源释放策略。
- 网络 IO 必须考虑超时、断连、重试和背压。

## 错误与日志

- 业务错误要有稳定错误码，不直接把底层异常暴露给上层。
- 日志必须携带请求标识、关键对象标识和失败原因。
- 不输出敏感信息。

## 测试与验证

至少按改动范围执行：

- 编译
- 单元测试
- 必要时 ASAN / TSAN
- 必要时 benchmark

如无法运行，必须说明未验证范围和剩余风险。

## 交付要求

最终说明必须包含：

- 改动内容
- 涉及模块
- 接口 / 协议影响
- 验证结果
- 剩余风险
