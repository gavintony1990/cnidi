# Bugs Memory

本文件记录跨任务、重复出现或高概率再次踩到的坑点。不要记录一次性问题，也不要记录已经彻底失效的临时上下文。

## 使用规则

- 只记录会重复出现的坑。
- 记录时说明：
  - 现象
  - 触发条件
  - 风险
  - 推荐规避方式
- 如果某条坑点已经不再适用，应及时删除。

## 当前已知长期风险

### 1. 文档命名与实际目录不一致会让 Agent 漂移

- 现象：规则和提示词引用的目录名与实际目录不一致，导致实现路径判断错误。
- 风险：需求归属、接口契约、规则适用范围都会错位。
- 规避：所有治理文档必须与实际目录名保持一致。

### 2. 只写规则不写模板，框架会停留在“描述层”

- 现象：有 rules 和 agents，但没有 docs、task 模板、示例工程时，协作路径无法闭环。
- 风险：Agent 无法稳定落地，团队也难以复用。
- 规避：优先补齐 docs、rules 索引、task 模板和工程骨架。
