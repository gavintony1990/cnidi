# Vue Component Encapsulation Rules

本文件补充 Vue 组件封装的具体约束。

## 组合与拆分

- 优先通过组件组合和 slots 解决复用
- 只有在多页面都需要相同业务语义时，才抽业务组件
- 页面区块优先封装稳定结构，如筛选区、工具栏、编辑抽屉

## composables 使用

- composable 用于封装稳定复用的状态与副作用
- composable 不应绑死单个页面的复杂分支逻辑
- composable 返回值应清晰、稳定，不要把页面所有状态打包进去

## 表格与表单

- 表格列定义、筛选条件、批量操作逻辑集中管理
- 表单默认值、校验规则、提交流程放页面层或表单 composable
- 弹窗/抽屉中的编辑表单优先拆成独立业务组件

## 错误封装

- 组件只处理自身局部错误
- 请求错误、权限错误和全局错误优先在页面层或统一请求层收敛

## 反模式

- 单文件组件同时处理列表、表单、权限、请求和弹窗全部逻辑
- 一个 composable 返回过多无关状态和回调
- 通过巨型 props 对象隐式驱动组件所有行为
